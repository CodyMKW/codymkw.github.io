🌳 okay so what even is this

the elevator pitch, minus the elevator

Town Ledger is a browser-based save editor for Animal Crossing on the GameCube — specifically built to also play nice with the Deluxe mod by Cuyler36. You drag your .gci save file in, it reads out your player, your town, your villagers, all of it, you tweak whatever you want, and it spits back a save file you can drop right back onto your memory card or Dolphin.

No installs, no plugins, nothing gets uploaded anywhere — it's a single HTML file that runs entirely in your browser. Open it, drop a save in, and go.

🤔 why does this exist

Honestly? I already had a whole pile of GameCube-era save editors going (Breath of the Wild, Splatoon, Tomodachi Life, Mario Kart Arcade GP...) and Animal Crossing was the obvious next one to tackle. But the twist here is I don't just play the base game — I run the Deluxe mod, so I wanted something that wouldn't choke on a modded save, and that I could keep expanding as I figured more stuff out.

Every single offset in here — where your bells live, where your name is stored, how the checksum gets calculated, all of it — got pulled from actually cracking open a real save file byte by byte and cross-checking it against ACSE (Cuyler36's own editor, built off the game's decompilation). Nothing in the "safe" parts of this tool is a guess. If I couldn't verify something, I said so instead of pretending.

🧭 how to actually use it

three steps, zero technical knowledge required

  • Get your save file off your memory card. If you're on real hardware, you'll need a way to pull the .gci off (GCMM, a homebrew save manager, etc). If you're on Dolphin, it's sitting in your GCI folder already.
  • Drop it into the editor. Drag-and-drop onto the page, or click to browse. It'll tell you immediately if the file isn't a real GAFE/GAFP/GAFU save — no silent failures.
  • Edit stuff, then hit download. The checksum gets recalculated automatically and both save copies inside the file get kept in sync, exactly like the game does on its own. Just drop the new file back where the old one was.
seriously though, back up your original save first This should go without saying but I'm saying it: keep a copy of your save before you start messing with it. I've tested this thing pretty hard against my own real save data, but "pretty hard tested" and "bulletproof" aren't the same thing, and it's your town.

📑 what's on each tab

the guided tour

🙂 Player one of these per resident found in your save.

  • Profile — name, gender, face, birthday. Names use the actual in-game AC character set (matched against a 256-entry table), so accented characters and symbols the game supports will render right.
  • Mr. Resetti — your reset counter. Zero it out if he's been giving you an earful.
  • Wallet — bells on hand, your loan to Tom Nook, and your post office savings (these are three separate values, learned that one the hard way after a real in-game test).
  • Equipped & wallpaper — held item, shirt, and your Pockets menu's background design.
  • Pockets — all 15 inventory slots, searchable by item name instead of raw hex.

🏘 Town the town-level stuff.

  • Town name (syncs across every player and villager record automatically on download)
  • Weather — heads up, this doesn't force what you visually see when you load in, but setting it to Rain does mark your flowers as watered/sparkling, so it doubles as an accidental "water all flowers" button lol
  • Train station design — 15 looks, Wooden/Metal/Brick in five variants each
  • 🌿 Remove all weeds — scans all 30 acres and clears them out in one click
  • 🧽 Clean up stale name copies — fixes a weird undocumented leftover field that can hang onto an old, mistyped version of your name even after you've fixed it properly elsewhere

🐾 Villagers your 15 resident lots.

  • Swap species (searchable, ~237 base-game villagers), personality, catchphrase, and shirt
  • Empty lots show as empty — moving someone in isn't supported, that needs real house placement this tool doesn't set up

🔍 the item search thing

Every item field — held item, shirt, wallpaper, all 15 pockets, villager shirts — is a live searchable box instead of a raw hex field. Type a name, or the hex ID itself, and pick from the dropdown.

The database started at about 2,300 items from one community list, then got swapped for a bigger ~3,000-entry one, then had roughly 1,100 unsafe entries (terrain objects, buildings, corrupt/warp-zone stuff — things that have no business sitting in a pocket) manually stripped back out. What's left is 1,976 items that are actually safe to carry around.

found a wrong ID? The list isn't 100% perfect — if the editor shows the wrong item name for something in your save, message me on Bluesky and tell me what it actually is in-game. I'll fix the entry.

🚫 what's deliberately left out

not laziness, just not willing to guess with your save file

Deluxe adds real features on top of the base game — town ordinances, extra villagers, probably more I don't even know about — but Deluxe isn't open source, so there's no reference material to check any of that against. Everything else in this editor got cross-verified against the game's actual decompilation before it shipped; Deluxe's additions can't be, so most of it stays out entirely rather than me inventing an offset and hoping.

Also skipped, for similar reasons:

  • Watering flowers directly — even the reference tooling this whole editor is built on top of doesn't implement this for the GameCube version, only later games. The Rain-weather trick above gets you most of the way there anyway.
  • Native fruit — the byte where this should live doesn't match the documented format at all in Deluxe saves, which suggests something got shifted around. Not touching it blind.
  • Moving villagers in/out of lots — needs real house coordinate setup, out of scope.
known quirk: villager species swaps Swapping who lives in a lot works great outdoors — they'll walk around town totally normally — but there are reports of the new resident getting stuck at their own front door and not actually going inside. House interior data for villagers isn't documented anywhere I could find, so I genuinely don't know what's causing it yet. Treat species swaps as an outdoor-appearance thing for now.

🔬 the "how do I know this actually works" section

A few of the trickier fields in this editor didn't come from a reference doc at all — they came from actual before/after testing against a real save. A couple of favorites:

  • The post office savings field was originally at the wrong offset entirely. Found the real one by depositing exactly 50,000 bells in-game, re-exporting the save, and searching the whole file for that exact number. Found it 12 bytes later than the earlier guess.
  • The stale name field got found the same way — noticed a leftover space-prefixed copy of a name floating around in an undocumented spot inside villager records, confirmed it existed in both occupied and completely empty lots (a sign it's leftover junk, not something the game actively reads), and built a cleanup tool that only touches it when it's a confident match.

Basically: wherever a public reference existed, it got checked against that. Wherever it didn't, it got checked against reality instead of guessed at.

❓ quick questions

Does this work with an unmodded save?

Yep — Deluxe support was the whole reason it got built, but a totally vanilla AC:GCN save works exactly the same way.

Is my save file uploaded anywhere?

No. Everything happens locally in your browser. Nothing gets sent to a server.

Will this work on real GameCube hardware, or just Dolphin?

Both — it's just editing the standard .gci save format, so as long as you can get the file on and off your memory card, it doesn't matter how you're playing.

An item shows the wrong name, what do I do?

Message me on Bluesky with the ID and what it actually is, and I'll get it corrected.

Will Deluxe-specific features ever get added?

Maybe, if I can find a reliable way to verify offsets for them — most likely through careful before/after save comparisons, the same way a couple of the trickier base-game fields got nailed down.
